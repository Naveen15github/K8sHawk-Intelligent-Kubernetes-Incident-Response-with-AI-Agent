# K8sHawk Knowledge Base

This directory stores all incident documentation and voice alerts generated by K8sHawk.

## Directory Structure

```
knowledge-base/
├── rca/                    # Root Cause Analysis documents
│   ├── 2026-04-16/        # Organized by date (YYYY-MM-DD)
│   │   ├── rca_pod-name_20260416_143022.md
│   │   └── rca_deployment-name_20260416_150145.md
│   └── 2026-04-17/
│       └── ...
│
└── voice-messages/         # Voice alert audio files
    ├── 2026-04-16/        # Organized by date (YYYY-MM-DD)
    │   ├── voice_pod-name_20260416_143022.wav
    │   └── voice_deployment-name_20260416_150145.wav
    └── 2026-04-17/
        └── ...
```

## File Naming Convention

### RCA Documents
- **Format**: `rca_{resource-name}_{timestamp}.md`
- **Example**: `rca_order-service_20260416_143022.md`
- **Timestamp**: `YYYYMMDD_HHMMSS` format

### Voice Messages
- **Format**: `voice_{resource-name}_{timestamp}.wav`
- **Example**: `voice_order-service_20260416_143022.wav`
- **Timestamp**: `YYYYMMDD_HHMMSS` format
- **Audio Format**: WAV (22.05 kHz, mono)

## Date Organization

All files are automatically organized into date-based folders:
- **Format**: `YYYY-MM-DD` (e.g., `2026-04-16`)
- **Purpose**: Easy navigation and cleanup of old incidents
- **Retention**: Configure retention policies based on your needs

## GitHub Integration

### What's Tracked
- ✅ Folder structure (empty folders with `.gitkeep`)
- ✅ README documentation

### What's Ignored
- ❌ RCA markdown files (`*.md`)
- ❌ Voice audio files (`*.wav`, `*.mp3`)

This keeps your repository clean while maintaining the folder structure for easy setup.

## Audio Format Details

Voice messages are stored in WAV format with the following specifications:
- **Sample Rate**: 22,050 Hz
- **Channels**: Mono
- **Bit Depth**: 16-bit
- **Codec**: PCM (uncompressed)
- **File Size**: ~2-3 MB per minute

### Why WAV Format?
- ✅ Universal compatibility
- ✅ High quality audio
- ✅ No licensing issues
- ✅ Easy to process and upload to Slack
- ✅ GitHub-friendly (though ignored in commits)

## Usage

### Accessing RCA Documents
```bash
# View today's RCAs
ls knowledge-base/rca/$(date +%Y-%m-%d)/

# Search for specific incident
find knowledge-base/rca -name "*pod-name*"
```

### Playing Voice Messages
```bash
# Play latest voice message
.\scripts\utils\play-latest-voice.ps1

# Play specific voice message
ffplay knowledge-base/voice-messages/2026-04-16/voice_pod-name_20260416_143022.wav
```

### Cleanup Old Files
```bash
# Remove RCAs older than 30 days
find knowledge-base/rca -type f -mtime +30 -delete

# Remove voice messages older than 7 days
find knowledge-base/voice-messages -type f -mtime +7 -delete
```

## Configuration

Storage paths are configured in `k8shawk/config/settings.py`:

```python
# Storage Configuration
rca_dir: Path = Path("knowledge-base/rca")
voice_dir: Path = Path("knowledge-base/voice-messages")
```

You can override these via environment variables:
```bash
export RCA_DIR="custom/path/to/rca"
export VOICE_DIR="custom/path/to/voice"
```

## Backup Recommendations

### For Production
1. **RCA Documents**: Back up to S3/Azure Blob/GCS
2. **Voice Messages**: Optional (can be regenerated)
3. **Retention**: Keep 30-90 days based on compliance needs

### For Development
- RCA and voice files are gitignored
- Folder structure is preserved
- Easy to recreate on new clones
